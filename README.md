# 🔥 FireDash — Система мониторинга устройств

FireDash — это кроссплатформенное решение для мониторинга состояния устройств. Состоит из клиента, который собирает метрики, и сервера, который принимает данные, сохраняет их и предоставляет веб-интерфейс для просмотра в реальном времени.

📦 Состав проекта

Проект состоит из двух основных компонентов:

1. FireDashClient — клиентская программа:
   - Работает в фоне;
   - Автоматически добавляется в автозагрузку;
   - Сбор данных о CPU, GPU, батарее, аптайме и активных процессах;
   - Отправляет информацию на сервер каждые 60 секунд.

2. FireDash API / Server — серверная часть на базе FastAPI:
   - Принимает и сохраняет данные от клиентов;
   - Инициализирует базу данных;
   - Отдаёт статический интерфейс для просмотра данных.

⚙️ Как это работает
🔁 Клиент:
При запуске:

Проверяет, установлен ли флаг installed.flag.

Если нет — копирует себя в автозагрузку (в зависимости от ОС).

Запускается в фоне (отключает консоль).

Каждую минуту:

Считывает информацию о:

CPU загрузке;

GPU загрузке (через GPUtil);

Заряде батареи;

Аптайме;

3 самых нагруженных процессах.

Отправляет данные POST-запросом на сервер.

🌐 Сервер:
При старте создаёт базу данных.

Роуты:

/api/logs/ — API для приёма логов.

/dashboard — отображает HTML-интерфейс (из app/frontend).

Использует FastAPI + StaticFiles для отдачи клиентской части.
